Git的简介与集中式的区别

Git是目前世界上最先进的分布式版本控制系统，没有之一，与集中式的版本控制系统的不同是，集中式版本控制系统的版本库是集中存放在中央服务器的，对于集中式的弊端深有体会，网络一但出现故障，集中式就歇菜了，中央服务器一出问题，也歇菜了。而对于分布式来说每个人都是版本库，做出修改后相互推送就可以看到做出了什么修改，其安全性也比集中式高，比如其中一个人电脑坏了也不要紧，因为每个人都是版本库，随便去别的电脑复制一份就好了。

Git基本知识

1.在上节中提交了一个xxx文件，现在修改文件的内容后，执行git status，可以让我们时刻掌握仓库当前的状态，命令输出告诉我们，xxx文件被修改过了，但还没有准备提交的修改

2.当我们忘记上次对文件进行过什么修改的时候，我可以可以执行 git diff命令，顾名思义就是查看difference，显示的格式正是Unix通用的diff格式，可以看出我们修改的是什么

3.再次提交代码，git add xxx，git commit -m “消息”

版本回退

1.我们平时工作写代码写完某个模块，习惯性的按下c+s保存，git也是，当我们修改到一定的程度的时候，可以将其保存为快照（commit）,我们难免会遇到把文件改乱了或者是误删了，这个时候，我们就可以从最近的快照中恢复继续工作，在git中我们可以使用git log命令查看我们提交的历史版本，显示从最近到最远的提交日志，我们可以看到3次提交，最近的一次是修改第二行，上一次是在第一行添加了distributed，最早的一次是wrote a test file。如果嫌输出信息太多，看得眼花缭乱的，可以试试加上--pretty=oneline参数，出现的字符串为版本号，使用的是SHA1计算的，现在我们回退到上一个版本，执行git reset -- hard HEAD^就可以回退到上一个版本了，HEAD指向的版本就是当前版本，但是执行git log会发现回退之前的版本已经不在了，我们要想回去又该如何，只要你窗口没关，就可以找到上次历史版本的版本号进行回退git reset --hard 9101b7eb2e6803a409115df05efb8e256e0c8a4f当然，就算我们关闭了窗口我们也可以执行git reflog记录你的每一次命令

工作区和暂存区

1.工作区：就是我们一开始创建的文件夹，这个就是工作区
2.版本库：在工作区中有一个.git隐藏目录，称为版本库，Git的版本库里面存放了很多东西，其中最为重要的是stage,也就是暂存区，之前我们提交文件到Git版本库的时候执行git add xxx就是添加到暂存区，之后执行git commit -m就是将暂存区的所有文件提交到当前分支，在我们创建版本库的时候，git会自动帮我们创建一个master分支

管理修改

Git之所以比其他版本控制系统优秀，是因为Git跟踪并管理的是修改而并非文件，为何这么说，假设我们现在修改一个文件，执行git add xxx命令，之后再去修改文件，直接提交，我们第一次修改 执行git add之后就将文件提交到了暂存区，而第二次修改并未提交，Git针对的是修改，而并非文件，不管进行多少次修改，你并没有将其提交到暂存区，他都不会提交不在暂存区的修改，所有每次修改之后就行一次git add在执行git commit

撤销修改

如果你不小心将工作区的某个文件内容修改错误了，你想回退，删除这个修改，可以执行git checkout -- file回退到你修改之前的版本，假设你又不小心添加到了缓存区，你可以执行git reset HEAD<file>，之后在执行git checkout -- file就可以回退到自己未修改前的版本了

删除文件

在Git中我们在工作区添加了一个文件并进行提交，之后你删除了该文件，我们决定要删除，就可以从版本库中删除该文件，执行git rm xxx以及git commit 就从版本库中删除了，如果我们返回了不想删除，就可以执行git checkout -- file就可以还原文件了

添加远程库

在添加之前，我们应该让本地Git仓库和GitHub远程仓库之间通过ssh通信：
（1）创建SSH Key，在用户主目录下查找.ssh目录找到id_rsa.pub文件复制里面的内容
（2）登录GitHub，打开setting，点击ssh key进行添加
然后我们就在GitHub上创建一个仓库，然后执行git remote add 远程库名 git@github.com:git账户名/本地仓库名.git之后我们就可以将本地仓库的所有内容推送到远程库上去，执行git push -u 远程库名 master,现在只要本地提交了，就可以将最新修改推送到GitHub上，执行git push 远程库名 master

远程库克隆

登录GitHub，创建远程库，执行git clone git@github.com:github用户名/远程库名.git
